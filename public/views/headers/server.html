Who has a need to know what type of server you're running? What It Looks Like: Server: Apache-Coyote/1.1

Tell Me More

A lot of what is said on our X-Powered-By page applies here, which boils down to not telling anyone about how your site works unless they really need to know how your site works. The more people know about how your site works, the more information they have about what vulnerabilities might exist on your site.

In the example header shown above, the it's revealed that Apache-Coyote is running. Here are some other examples that are sometimes disclosed:

  • server: IBM_HTTP_Server
  • Server: Apache/2.2.17 (Fedora)
  • Server: nginx/1.2.1
  • Server: Apache/2.2.23 (Unix) mod_ssl/2.2.23 OpenSSL/0.9.8e-fips-rhel5 mod_auth_passthrough/2.1 mod_bwlimited/1.4 FrontPage/5.0.2.2635
In my humble opinion, there's really no need to include any of that stuff in one's headers. Let me know if you feel otherwise. So the recommendation is to remove Server: from your headers.

Apache Makes It Hard

Apache doesn't make it easy to remove the entire reference to the Apache server in one's headers. That is, you can remove the Apache version number but it still works hard to keep Server: Apache in there.

Does This Really Help?

Some might say that removing the reference to the server and possibly its version number in one's HTTP headers is not all that useful because there are other ways to "fingerprint" a server. By poking, prodding, and pinging servers in certain ways and watching the responses you can deduce what server type and maybe even version is running. But making an attacker work harder is always preferable, so it's still recommended to attempt to remove refererences to the server type and version from one's headers.
